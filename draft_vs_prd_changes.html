🆕 D. 신규 추가 항목
15
2가지 진입 경로 (Path A / Path B) 추가 신규
❌ 드래프트
단일 경로: 구독 가입 → 포토북 생성
(드래프트에 "처음 경험: Phase 2 → Phase 1" 언급은 있으나 별도 플로우 없음)
✅ PRD
Path A: 표준 구독 가입 플로우 (가입 → 생성)
Path B: 마케팅 진입 — 포토북 생성을 먼저 경험 후 발행 시 결제 트리거
프로토타입에 두 경로 모두 셀렉터로 구현됨
Path B는 "먼저 경험하고 결제" 모델. CVR 최적화를 위한 마케팅 캠페인 활용 목적.

"내부 AI 사진 분석"
"중복/불량 제거 · 얼굴/장소/이벤트 감지"
구체적 파이프라인 미정

2단계 파이프라인 확정:
1) On-device photo clustering (디바이스 내 사전 그룹핑)
2) Server-side AI curation (서버 큐레이션으로 30장 선정)

디바이스에서 사전 클러스터링 후 서버로 전송하여 네트워크 비용 절감. 추가 레퍼런스: Chatbooks, Klikkie, Mootsh.
17
프로토타입 OS 분기 (iOS/Android) 신규
❌ 드래프트
OS별 분기 미고려
iOS/Android 동일 플로우로 기술
✅ PRD 프로토타입
5가지 플로우로 분기:
1) 최초가입자 × iOS
2) 최초가입자 × Android
3) 정기사용자 × iOS
4) 정기사용자 × Android 즉시실행(foreground)
5) 정기사용자 × Android 배치(background, V2)
사진 업로드 방식이 OS별로 다르므로 프로토타입 단계에서 분기 확인 가능하도록 셀렉터 구현.
⏳ E. 여전히 미정 — 논의 필요 항목
드래프트에서 TBD였고, PRD 작업 과정에서도 아직 최종 확정되지 않은 항목들입니다.
미사용 월 처리 정책 — 사진이 없는 달의 구독료·포토북 생성 처리 방안
해지 후 디지털 포토북 접근 — 해지 후 기존 포토북 열람 기간 (프로토타입에는 "6개월" 표기, 최종 미확정)
포토북 프로젝트 삭제 정책 — 사용자가 포토북을 삭제할 수 있는지 여부
시즌 요금제 실물 생산 대상 월 — 분기 중 어느 달 포토북을 실물로 제작할지
커버 대표 사진 선정 기준 — AI가 자동 선정 vs 사용자 선택
선물하기 기능 범위 — V1 스코프 제외 유지 여부
포토북 생성 트리거 — 월 1회 고정일 자동 vs 사진 누적 기반
실물 배송 방식 — 자동 발주 vs 고객 최종 확인 후 발주
제품 사양 — 사이즈, 페이지 수, 커버 재질, 책등 포함 여부
해지 리텐션 오퍼 정책 — 할인율, 무료 연장 기간 등 구체 내용
🔧 개발 확인 필요 사항
PRD 작업 중 도출된 기술 확인 포인트입니다.
한 달치 썸네일 전체 업로드 후 큐레이션 시작 여부 — 전체 업로드 완료 후 30장 선정 시작하는지, 점진적 처리 가능한지
프리뷰 렌더링과 전체 업로드 병렬 처리 여부 — 프리뷰를 먼저 보여주고 원본 업로드를 백그라운드로 진행할 수 있는지

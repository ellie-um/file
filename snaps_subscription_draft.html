전체 흐름 (Overview)
Phase 1

🔑 구독 가입

  • 요금제 선택 (무료 체험 포함)
  • 결제 등록
  • 커버 타입 선택 + 공유 대상자 등록
  • 사진 접근 권한 동의
Phase 2

📸 디지털 포토북 자동 생성

  • 사진 자동 분석 (내부 AI)
  • 사진 추천·큐레이션
  • 디자인 자동 구성 + 커버 커스텀
  • 사용자 확인 후 발행
Phase 3

📦 실물 배송

  • 요금제별 실물 배송 분기
  • POD 제작 발주
  • 구독자 주소 자동 적용
Phase 4

🔗 공유 & 수신자 주문

  • 공유 대상자 자동 발송
  • 수신자 비회원 열람

📋 핵심 정책
  • 1아키텍처: 회원/DB는 스냅스 공유, 서비스 로직은 독립 모듈. 추후 별도 앱 분리 시 최소 수정 구조 (네이버/네이버 스토어앱 모델)
  • 2요금제: 무료 체험 / 디지털 Basic / 실물 포함 Standard / 실물 프리미엄 — 3~4 티어, 상세 가격 협의 필요
  • 3생성 주기: 월 1회 자동 생성 (고정일 or 사진 N장 누적 기준, 협의 필요)
  • 4실물 배송: 자동 발주 OR 고객 최종 확인 후 발주 (협의 필요)
  • 5커버 커스텀: 색상 5~6색 팔레트 + 타이틀 1줄 (최대 20자) + 날짜 자동 표기. 실시간 미리보기. 책등은 사양 확인 후 결정 (P2)
  • 6공유 & 수신자: 수신자 목록 사전 등록 → 생성 시 자동 발송 (카카오 알림톡 우선). 수신자 실물 주문 시 계정 가입 여부 협의 필요
  • 7플랫폼 전략: 모바일 앱 메인. PC는 안내 + 내역 조회 + 앱 다운로드 유도만 제공
⚙️ 아키텍처 고려사항 스냅스 회원 연동 (SSO) + 독립 서비스 도메인 분리 가능 구조 → 서비스 DB 분리, API 게이트웨이 설계, 추후 별도 앱 패키지로 분리 가능하도록 초기 설계 반영 필요
🤝 협의 필요 항목
  • ⚠️요금제 구성 및 가격 정책 — 티어 수, 가격대, 무료 체험 기간·제한 조건
  • ⚠️포토북 생성 트리거 기준 — 월 1회 고정 날짜 vs 사진 N장 이상 누적 시 vs 하이브리드
  • ⚠️실물 배송 방식 — 자동 발주 (고객 확인 없이) vs 고객 최종 확인 후 발주
  • ⚠️커버 책등 포함 여부 — 제작 사양 및 원가 확인 후 결정
  • ⚠️수신자 주문 시 계정 가입 필수 여부 — 비회원 주문 허용 범위
  • ⚠️포토북 상품 스펙 확정 — 사이즈, 내지, 페이지 수, 커버 책등 포함 여부 통합 확정
  • ⚠️스냅스 앱 내 UX 정합성 — 기존 포토북 편집기와의 UI 일관성 전략
  • 🔥해지 리텐션 오퍼 정책 신규 — 해지 시도 시 제공할 할인율/무료 연장 기간, 리텐션 CTA 노출 조건, 일시정지 옵션 제공 여부
  • 🔥시즌 요금제 실물 배송 기준 신규 — 가입 시점 기준 첫달/3개월차 판정 기준 (예: 3월 가입 시 첫 실물 발송 시점)
  • 🔥사용자 검토/수정 범위 신규 — 자동 생성된 포토북에 대해 확인만 가능 vs 수정 허용 범위 결정
  • 🔥구독자 선물하기 기능 신규 — 구독자가 수신자에게 실물 선물 결제 기능 제공 여부 및 플로우
